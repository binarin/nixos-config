# Cache-only NativeLink Bazel cache on `docker-on-nixos` — design

**Date:** 2026-07-06
**Status:** design, pending implementation
**Scope:** experimentation — a shared, cache-only NativeLink (remote cache, no remote
execution) backed by Redis (hot) + Garage S3 (durable), deployed as a single Arion
project on the `docker-on-nixos` host and exposed over Tailscale.

## Goal

Stand up a working Bazel **remote cache** so builds can share action results and
output blobs, using NativeLink in **cache-only** mode. This is an experiment: prove
the data path (Bazel → NativeLink → Redis/Garage) end to end with a throwaway local
Bazel project. No CI wiring, no remote execution.

## Background / key facts established during design

- **NativeLink is not in nixpkgs.** The upstream repo is checked out at
  `~/personal-workspace/nativelink` and its `flake.nix` (flake-parts) exposes, among
  others:
  - `packages.<system>.nativelink` — the server binary (static musl on Linux). One
    binary is the whole cache/executor.
  - `packages.<system>.nativelink-image` — a nix2container OCI image (built from
    source, not a ghcr pull).
  - `flakeModules.nixos` — **not** a service module; it only generates a client-side
    `nixos.bazelrc`. There is no upstream NixOS module to run the server.
- **Two caches, one protocol.** NativeLink multiplexes CAS, Action Cache (AC),
  ByteStream, Capabilities, and Health on a **single gRPC/HTTP2 listener**
  (`0.0.0.0:50051`). Cache-only = keep `cas`/`ac`/`bytestream`/`health`, drop
  `execution` + scheduler.
- **Garage is S3-compatible** and already deployed (`modules/machines/garage.nix`),
  region `garage`, reachable at `s3.lynx-lizard.ts.net` (Tailscale serve → the host's
  `:3900` S3 API; `:3900` is **not** opened on the LAN firewall).
- **S3 client = official aws-sdk-rust** (`aws-sdk-s3 = 1.82.0`,
  `aws_config::defaults(BehaviorVersion::latest())`). Therefore:
  - Custom endpoint (Garage/MinIO) is set via **`AWS_ENDPOINT_URL`** env — the `aws`
    provider struct has no endpoint field by design.
  - The SDK's default flexible-checksum behaviour (`WHEN_SUPPORTED`, CRC32 on
    `PutObject`) is **rejected by Garage**. Fix already proven in this repo
    (`tools/ncf/ncf/commands/aws_env.py`): set
    `AWS_REQUEST_CHECKSUM_CALCULATION=WHEN_REQUIRED` and
    `AWS_RESPONSE_CHECKSUM_VALIDATION=WHEN_REQUIRED`.
- **Tailscale free-plan machine cap** → we expose the cache as a Tailscale **service**
  (`tailscale serve`) rather than standing up a dedicated tailnet node. The
  `bazel-cache` service uses `protocol = "tls-terminated-tcp"` on `:443` forwarding to
  `localhost:50051` — raw byte forward, which carries gRPC/HTTP2 correctly and
  sidesteps any h2c-through-HTTPS-reverse-proxy issues.

## Architecture

```
Bazel client
   └─ grpcs://bazel-cache.<tailnet>:443
        │  (tailscale serve: tls-terminated-tcp → localhost:50051, raw TCP)
   docker-on-nixos
   ┌───────────────────── arion project "nativelink" ─────────────────────┐
   │  service nativelink  ──(image.contents from flake binary)             │
   │      │  listens 0.0.0.0:50051 (h2c: cas/ac/bytestream/health)         │
   │      ├─ redis://redis:6379  ─────────►  service redis (private bridge,│
   │      │                                   no host port published)      │
   │      └─ AWS_* env (SigV4) ──────────►  Garage S3                       │
   └───────────────────────────────────────────────────────────────────────┘
                                              region "garage"
                                              bucket "nativelink-cache"
                                              endpoint https://s3.lynx-lizard.ts.net
```

- **One Arion project, two services, private bridge.** Redis publishes **no host
  port** → visible only to NativeLink.
- **NativeLink image built by Arion** from the flake binary (`image.contents`), no
  ghcr pull, no `docker load` step.
- **State lives entirely in Redis + Garage.** The NativeLink process is stateless.

## Components

### 1. Flake input (first implementation step → reload checkpoint)

Add, in the new module, via `flake-file` (github:vic/flake-file, dendritic):

```nix
flake-file.inputs.nativelink.url = "github:TraceMachina/nativelink";
# NOTE: deliberately NO `inputs.nixpkgs.follows` — NativeLink's musl/rust cross
# build is sensitive; let it use its own pinned nixpkgs so the binary builds.
```

**STOP after this change.** `flake.nix` is auto-generated by `flake-file`; the user
reloads / re-locks before implementation continues. This is an explicit checkpoint.

### 2. NixOS module `modules/services/nativelink.nix`

Follows the dendritic + arion pattern of `modules/services/karakeep.nix`:

- `flake.nixosModules.nativelink`, `key = "nixos-config.modules.nixos.nativelink"`.
- Imports: `arion`, `sops-nix`, `tailscale`.
- Imported into `docker-on-nixos-configuration` alongside the existing services.

Arion project (`virtualisation.arion.projects.nativelink`, `backend = "docker"`):

- **service `nativelink`**
  - `image.contents = [ inputs.nativelink.packages.x86_64-linux.nativelink pkgs.cacert ]`
    (arion builds + loads the image; `cacert` needed for TLS to Garage).
  - `image.command` / service `command` = run `nativelink` against the mounted config.
  - Config file mounted read-only (see §4). **No secrets in the config.**
  - `env_file` = the secret env-file (see §3).
  - Attached to the project's private bridge network only. Container port `50051`
    published to `127.0.0.1:50051` on the host (so `tailscale serve` can reach it) —
    **not** to `0.0.0.0`.
  - `depends_on = [ "redis" ]`.
- **service `redis`**
  - `image` = pinned Redis (tag tracked in a `nativelink.json` tags file, mirroring
    `karakeep.json`).
  - Single-node, **no host port published**, private bridge only.
  - Ephemeral is acceptable (hot tier over durable S3); no persistent volume required
    for the experiment.

### 3. Secrets — Garage credentials → arion env-file

Chosen approach: **`sops.templates` env-file**, matching `modules/services/karakeep.nix`
on the same host.

- A dedicated Garage access key + secret for NativeLink (see §5 prerequisites),
  provisioned once on the Garage side and stored as sops secrets on the
  `docker-on-nixos` host: `nativelink/s3-access-key` + `nativelink/s3-secret-key`.
- `sops.templates."nativelink-env"` renders the env-file from those placeholders,
  with `restartUnits = [ "nativelink-docker-compose.service" ]`. Arion consumes it via
  `env_file`. The env-file contains:

  ```
  AWS_ACCESS_KEY_ID=<garage nativelink key-id>
  AWS_SECRET_ACCESS_KEY=<garage nativelink secret>
  AWS_DEFAULT_REGION=garage
  AWS_ENDPOINT_URL=https://s3.lynx-lizard.ts.net
  AWS_REQUEST_CHECKSUM_CALCULATION=WHEN_REQUIRED
  AWS_RESPONSE_CHECKSUM_VALIDATION=WHEN_REQUIRED
  ```

- All six vars are rendered into the single `sops.templates."nativelink-env"` file
  (the static ones are not secret but live there for simplicity / one `env_file`).

### 4. NativeLink config (cache-only), rendered by Nix

Rendered with `pkgs.writeText` as json5, based on `nativelink-config/examples/redis.json5`
(already cache-only) plus an `aws` slow tier from `s3_backend_with_local_fast_cas.json5`:

- `CAS_MAIN_STORE` = `existence_cache( compression( lz4, fast_slow( fast: redis, slow: aws-s3, key_prefix "cas/" ) ) )`
- `AC_MAIN_STORE`  = `completeness_checking( fast_slow( fast: redis, slow: aws-s3, key_prefix "ac/" ), cas: CAS_MAIN_STORE )`
- Redis store: `redis://redis:6379/`, **single-node / standalone** (not `cluster`).
- `experimental_cloud_object_store`: `provider: "aws"`, `region: "garage"`,
  `bucket: "nativelink-cache"`. No endpoint field (comes from `AWS_ENDPOINT_URL`).
  `insecure_allow_http` **false** (endpoint is https).
- `servers[0].listener.http.socket_address = "0.0.0.0:50051"`, `services` =
  `cas` + `ac` + `bytestream` + `health`. **No `schedulers`, no `execution`,
  no `capabilities`-with-scheduler** → cache-only.

### 5. Garage prerequisites (one-time, manual, on the Garage host)

```
garage bucket create nativelink-cache
garage key create nativelink
garage bucket allow --read --write nativelink-cache --key nativelink
```

Store the key-id/secret as the `nativelink/s3-access-key` + `nativelink/s3-secret-key`
sops secrets on `docker-on-nixos` (§3).

### 6. Tailscale exposure

`bazel-cache` is **already allocated as a Tailscale Service** (vip-service). This host
registers as a backend for it via the repo's `services.tailscale.serve` module. When a
second instance registers the same service elsewhere, Tailscale load-balances across
both automatically (see Statelessness & HA).

```nix
services.tailscale.serve.services.bazel-cache = {
  serviceName = "bazel-cache";
  protocol   = "tls-terminated-tcp";
  target     = "localhost:50051";
  # port 443 on the tailnet service
};
```

Redis is **not** exposed. (If remote `redis-cli` inspection is ever wanted, it would
be a separate `tls-terminated-tcp` service — explicitly out of scope here.)

## Client — local throwaway smoke test

A throwaway Bazel workspace (e.g. a trivial `genrule`/`cc_binary`) with:

```
# .bazelrc
build --remote_cache=grpcs://bazel-cache.<tailnet>.ts.net
build --remote_instance_name=main
```

Acceptance:
1. First `bazel build //...` populates the cache (writes to Redis + Garage).
2. `bazel clean` (and drop the local action cache), then rebuild → outputs served
   from the remote cache (all remote cache hits).
3. Confirm blobs land in the `nativelink-cache` Garage bucket (`cas/`, `ac/` prefixes).

## Statelessness & HA (mechanism already in place)

The cache tier holds **no node-local durable state** (memory + Redis fast tiers, S3
durable slow tier). Consequences:

- A NativeLink replica can be killed/restarted with **no data loss** — only a
  cold-cache perf dip while the fast tier re-warms from S3.
- **HA is already wired at the tailnet layer.** `bazel-cache` is allocated as a
  Tailscale **Service** (vip-service). It advertises **one backend today** (this
  host). The moment a **second** NativeLink instance is served from anywhere and
  advertises the same `bazel-cache` service, **Tailscale load-balances across the
  backends automatically** — no client change, no data-plane change, because the tier
  is stateless over shared S3+Redis. Adding a replica *is* the HA story.
- A **single backend is a compute SPOF today** (cheap: data survives in S3, recovery =
  restart). We run one replica for the experiment; scaling out is purely additive.
- **Redis caveats for a future multi-replica world:** a single shared Redis becomes
  the SPOF/bottleneck; you'd cluster it or drop it and let S3 + in-memory carry the
  hot path. Also **verify `fast_slow` degrades gracefully when the fast (Redis) tier
  is down** (should fall through to S3 rather than erroring) before relying on
  Redis being disposable.

## Non-goals (YAGNI for the experiment)

Remote execution (workers/scheduler), sharding, mTLS/client-cert auth,
Prometheus/OTel wiring, CAS eviction/GC tuning, HA replicas, Redis persistence,
Forgejo-runner/CI wiring.

## Assumptions

- The NativeLink container can resolve and route to `s3.lynx-lizard.ts.net` (Tailscale
  MagicDNS reachability from inside the container). Declared reachable per design
  decision; if it turns out a bridged container can't reach the tailnet endpoint, the
  fallback is either host networking for the NativeLink service or opening Garage
  `:3900` on the home LAN and using `http://<garage-lan-ip>:3900` +
  `insecure_allow_http: true`.
- `docker-on-nixos` is `x86_64-linux` (confirmed) → use
  `inputs.nativelink.packages.x86_64-linux.nativelink`.

## Open items to resolve during implementation

1. Redis image tag/pin and whether any persistence is wanted (default: none).
2. `redis_store` standalone-mode field name/shape against the current config schema.
3. Path-style vs virtual-host S3 addressing against Garage via `AWS_ENDPOINT_URL`
   (default SDK path-style is expected to work; `AWS_S3_FORCE_PATH_STYLE` is the knob
   if bucket resolution misbehaves).

Resolved at design: secrets wiring = `sops.templates` env-file (§3).

## Implementation order (high level)

1. Add `flake-file.inputs.nativelink` → **STOP for reload/lock**.
2. Garage prerequisites (bucket/key) + clan vars generator for the key.
3. `modules/services/nativelink.nix`: arion project (nativelink via `image.contents`
   + redis), config json5, env-file, tailscale `bazel-cache` service; import into
   `docker-on-nixos`.
4. Deploy; verify container comes up and connects to Redis + Garage.
5. Local throwaway Bazel smoke test → confirm cache hits + blobs in Garage.

## As-built (final, verified 2026-07-06)

Implemented on branch `feat/nativelink-bazel-cache`; verified end-to-end:
`bazel build --remote_cache=grpcs://bazel-cache.lynx-lizard.ts.net` → `2 remote cache
hit` on a clean rebuild, blobs durable in Garage under `cas/` + `ac/`.

**Final data path**

```
bazel ──grpcs://bazel-cache.lynx-lizard.ts.net──► tailscale vip-service (protocol=tcp, raw passthrough :443)
                                                       │ → localhost:50051
   docker-on-nixos ── arion project "nativelink" ──────┤
     • nativelink (ghcr v1.5.2): terminates TLS in-process (tailscale cert → rustls h2 ALPN);
       cache-only CAS+AC = existence_cache(compression(fast_slow(redis, s3))) / completeness_checking(fast_slow(redis, s3))
     • redis: internal hot tier
                          │ aws provider, AWS_ENDPOINT_URL=http://<garage-tailscale-ip>:3900
                          ▼  path-style (IP endpoint), insecure_allow_http, checksums OFF
                     Garage S3  (region "garage", bucket nativelink-cache)
   nginx defaultListenAddresses pinned to the node LAN IP → frees the vip's :443
```

**Deviations from the original design above, and why (each discovered during bring-up):**

1. **ghcr image, not `image.contents` from a flake input.** Chosen to avoid a second
   nixpkgs evaluation; the flake input (and its follows) were dropped. Pinned
   `ghcr.io/tracemachina/nativelink:v1.5.2@sha256:4552f2…`.
2. **Secrets = clan vars, not a plain sops file.** `clan.core.vars.generators.nativelink-s3`
   emits an env-file; Arion reads the generator's `files.env.path` (niks3 pattern).
   The sops-template-over-clan-var bridge (`sops.placeholder."vars/…"`) is **not
   eval-safe** before `clan vars generate` runs, so it was abandoned.
3. **Cache-only server needs `capabilities: [{ instance_name: "main" }]`** (no
   `remote_execution`). Omitting it makes Bazel's `GetCapabilities` 404 → Bazel
   disables the cache. `bytestream` must be the array form (map form is deprecated).
4. **S3 to Garage: path-style + checksums off, via the `aws` provider + an IP endpoint.**
   Garage needs path-style (MagicDNS won't resolve `<bucket>.s3.lynx-lizard.ts.net`),
   and rejects the AWS SDK's default flexible checksum. v1.5.2's `aws` provider has no
   path-style toggle, but an **IP endpoint forces path-style**; and it honors
   `AWS_REQUEST/RESPONSE_CHECKSUM_CALCULATION=WHEN_REQUIRED` to drop the checksum.
   Endpoint = garage node's **tailscale IP** `:3900` (raw S3 API, plaintext over
   WireGuard). The `ontap` provider does path-style but **forces a SHA256 checksum
   Garage rejects** ("Invalid signature") — so it was rejected.
5. **Exposure: raw `tcp` passthrough + NativeLink-terminated TLS, not `tls-terminated-tcp`.**
   Tailscale's `tls-terminated-tcp` terminator doesn't negotiate `h2` ALPN, which
   gRPC-over-TLS requires (ALPN failure). NativeLink terminates TLS itself with a
   `tailscale cert` for the vip-service name (`nativelink-tls-cert.service` + weekly
   renewal timer); client uses `grpcs://`. Plaintext `grpc://` on `:443` was rejected
   as a non-starter (wrong for `:443`; Bazel threw `io exception`).
6. **nginx must not bind `0.0.0.0:443`.** Its wildcard bind (from the four
   `.binarin.info` reverse-proxy vhosts) swallowed the vip-service's `:443`. Fixed via
   `services.nginx.defaultListenAddresses = [ <node LAN IP> ]` in `docker-on-nixos.nix`
   (those vhosts are LAN-only; tailscale access is through separate vip-services).

**Known limitation (accepted for the experiment):** rotating the Garage key
(`clan vars generate`) does not auto-restart the container — the env-file path is
stable and there's no eval-safe restart-trigger for a clan-var's content. Restart
`nativelink-docker-compose.service` manually after a key rotation.
